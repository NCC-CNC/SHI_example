# Session Handoff: 2026-07-16

## Summary

First session on SHI_example. Set up the full project harness, defined the
project (an interactive web app for exploring the Species Habitat Index),
captured the design in docs and ADRs, and built and deployed the app through
milestone M3. The app is live and demonstrates the SHI concept end to end
(land cover -> Area of Habitat -> per-species scores -> aggregated index),
interactive over time, for three Canadian species.

## Current state

What works:

- Full pipeline is live and interactive: scrub 1993-2025, change the baseline,
  toggle connectivity and the overlap, hover any cell for its type/suitability.
- Every species and the overall index decline from baseline over the scenario;
  the baseline year scores 100. 52 tests pass; CI green; deploy automated.

Architecture:

- `src/engine/` pure SHI logic (no React), unit-tested. `src/data/` typed
  content (land cover palette, species crosswalks, the synthetic scenario as a
  base grid + cumulative dated events). `src/ui/` React components.

Known limits / simplifications (by design, see the model ADR):

- Synthetic data only; no real ESA CCI. No species range or elevation clipping.
  Quality folded into suitability. Simple-mean aggregation only. Grid is 10x10
  (engine is size-agnostic; scaling not yet exercised).
- Colors verified headlessly (server-render smoke test); no visual screenshot
  was taken in-session. A human look at the live app is the outstanding visual
  check.
