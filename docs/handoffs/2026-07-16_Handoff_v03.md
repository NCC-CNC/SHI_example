Save/share a scenario via URL (PR #37, closes #22)

- New pure module `src/ui/share-state.ts` encodes year, baseline, connectivity,
  overlap, and edits to a compact URL hash, omitting defaults (e.g.
  `#y=2010&e=D3-B45`). Edits reuse the single-char land cover codes.
- App seeds initial state from the hash and keeps the URL in sync via
  history.replaceState. Syncing is suspended during the guided tour.
- Out-of-range or malformed shared state is ignored; edits are clamped to the
  grid so a stale link cannot crash the engine.
- "Copy share link" button in the sidebar.
- Fix: the tour's on-close reset now only fires on an actual close, not on
  initial mount (it was wiping URL-seeded edits).
- Design note: docs/design/04_share_state.md.

Generated ~30x30 landscape grid (PR #40, closes #38)

- New `src/data/landscape.ts`: deterministic, seeded generator (mulberry32 +
  value noise, no runtime randomness). Organic base landscape (forest upland,
  shrub ecotone, grassland, wetland, lake, meandering river, crop patch) plus
  the 1993..2025 disturbances as region operations: a wandering developed
  corridor fragments the forest, the wetland dries to barren, grassland converts
  to crops.
- `scenario.ts` consumes it behind the unchanged `landCoverForYear` API.
- `GridView`: hairline gap, no rounding, so fills read as a continuous
  landscape; the SVG still scales to its column so the footprint is unchanged.
- Guided tour retuned: demo edits derived from the actual landscape (an intact
  forest cell for the pixel edit; the computed corridor for the restoration).
- Tests: new landscape.test.ts; scenario tests assert the mechanic rather than
  fixed 10x10 coordinates.

Tour step 6 highlight (PR #43, closes #42)

- Step 6 (Edit a pixel) now highlights the Set land cover box and names the tool
  in the step text.

Repo hygiene

## Current state

What works:

- Full pipeline live and interactive on the generated ~30x30 landscape: scrub
  1993-2025, change baseline, toggle connectivity and overlap (overlap on by
  default), hover any cell, set land cover to edit, read the before/after change
  impact, and copy a shareable link that restores the scenario.
- Guided tour drives the real app through all seven steps and clears on Done.
- 76 tests pass; CI green; deploy automated.

Architecture:

- `src/engine/` pure SHI logic (no React), unit-tested. `src/data/` typed
  content plus the new `landscape.ts` generator feeding `scenario.ts`. `src/ui/`
  React components, `share-state.ts`, `tour.ts`.

Verification this session used a real browser (Playwright + Chromium, installed
with --no-save so it is not a project dependency) to screenshot and drive the
app: the two-column layout, the share round-trip, the landscape at 1993/2025,
and the tour's restoration and step 6 highlight were all confirmed visually.

Known limits / simplifications (by design):

- Synthetic data only; no real ESA CCI, no range or elevation clipping. Quality
  folded into suitability. Simple-mean aggregation. Grid is a generated ~30x30
  on SVG (not 100x100; that would need a canvas renderer).
- The editable map still puts a tab stop on every cell (~900 at 30x30). Tracked
  as #41.
